# SizeMe Development Workflow

أنت تعمل على موقع SizeMe — متجر إلكتروني بملف واحد (`D:\sizeme\index.html`).
المستخدمون 95% على الجوال. كل تعديل مهما كان صغيراً يتبع الخطوات التالية بالترتيب.

---

## الخطوة 1 — اطلب الإذن قبل أي تعديل

قبل أن تلمس أي كود، اعرض على المستخدم ملخصاً واضحاً:

```
📋 ما الذي سأغيره:
• [وصف واضح لكل تغيير مقترح]

⚠️ ما الذي قد يتأثر:
• [أي أقسام أو وظائف قد تتأثر]

هل تريد المتابعة؟
```

لا تجري أي تعديل قبل موافقة صريحة ("نعم" / "اكمل" / "موافق" أو ما يعادلها).

---

## الخطوة 2 — قائمة فحص الجوال (قبل كتابة الكود)

تحقق من هذه النقاط قبل كتابة الكود:

| الفحص | المعيار |
|-------|---------|
| أحجام الخط | استخدم `clamp()` أو `var(--fs-*)` — لا أحجام ثابتة |
| أزرار اللمس | الحد الأدنى 44×44px — استخدم padding كافٍ |
| Overflow أفقي | لا `width` ثابت أكبر من 100vw — استخدم `max-w-full` أو `w-full` |
| مناطق Safe Area | للعناصر الثابتة: استخدم `env(safe-area-inset-*)` |
| الصور | `width:100%; height:auto` أو `aspect-ratio` ثابت لمنع CLS |

إذا كان التعديل يمس CSS أو layout، اذكر صراحةً كيف يبدو على شاشة 390px (iPhone).

---

## الخطوة 3 — نفّذ التعديل

استخدم أداة Edit أو Write لتعديل `D:\sizeme\index.html`.

- عدّل فقط ما وعدت به في الخطوة 1
- لا تغيير جانبي بدون إخبار المستخدم
- لا تعديل على `PRODUCTS_META` أو `TOTAL_PRODUCTS` إلا إذا طُلب صراحةً

---

## الخطوة 4 — تحقق من صور المنتجات (عند إضافة منتج)

إذا كان التعديل يشمل إضافة منتج جديد، تحقق من وجود الصورة فعلياً:

**بنية المجلدات:**
```
D:\sizeme\
├── imagestshirts\    ← تي شيرت  — اسم: {globalNum}-{seq}-{imgKey}.jpg
├── imagespolo\       ← بولو      — اسم: {catSeq}-{seq}-{imgKey}.jpg
├── imagesjeans\      ← بنطرون    — اسم: {catSeq}-{seq}-{imgKey}.jpg
├── imagestracksuit\  ← تراكسوت  — اسم: {catSeq}-{seq}-{imgKey}.jpg
└── imagesshirts\     ← قمصان    — اسم: {catSeq}-{seq}-{imgKey}.jpg
```

قبل تعديل `PRODUCTS_META`، تحقق من وجود الصورة:
- `{folder}/{prefix}-1-{imgKey}.jpg` موجودة على الأقل
- الاسم مطابق تماماً (حساس لحالة الأحرف وشرطاتها)

إذا الصورة غير موجودة: أخبر المستخدم بالاسم الصحيح المطلوب قبل المتابعة.

---

## الخطوة 5 — أمر git بعد كل تعديل

بعد الانتهاء، أعطِ المستخدم هذا الأمر جاهزاً للنسخ في VS Code Terminal:

```bash
cd /d D:\sizeme && git add -A && git commit -m "تعديل: [وصف قصير]" && git push
```

استبدل `[وصف قصير]` بوصف دقيق لما تم تعديله بالعربية.

**مثال:**
```bash
cd /d D:\sizeme && git add -A && git commit -m "تعديل: إضافة منتج Nike Tracksuit رقم 30" && git push
```

> 💡 الأمر `cd /d D:\sizeme` يضمن تشغيل git من المجلد الصحيح دائماً.

## ملاحظات تقنية دائمة

- **TOTAL_PRODUCTS** يجب تحديثه عند إضافة أي منتج
- **catSeq** مستقل لكل مجلد — لا علاقة له بالرقم العالمي للمنتج
- **category** إلزامي لكل منتج غير تي شيرت
- **jeans** يستخدم مقاسات الخصر (38-48)، باقي الأقسام نظام XL (2XL-7XL)
- اللغة الافتراضية عربية RTL — اختبر أي عنصر جديد في وضع RTL أولاً
- حجم الصفحة لكل قسم: `CAT_PAGE_SIZE = 12`
